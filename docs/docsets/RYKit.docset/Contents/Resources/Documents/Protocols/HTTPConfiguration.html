HTTPConfiguration

public protocol HTTPConfiguration

HTTP Provider is the protocol that defines an API and the relationship you want to have with it. You can define many providers in your app if you have different API (yours and external for instance)

  • baseURL

    The base URL of your API, the link that is common to every request

    Declaration

    Swift

    var baseURL: URL { get }
  • delegate Default implementation

    The delegate used by NSURLSession, can be overriden to manage unsupported cases

    Default Implementation

    The default delegate uses the default instance of SessionManager

    Declaration

    Swift

    var delegate: SessionManagerDelegate { get }
  • session Default implementation

    The session used by the provider. A default session is provided, but this can be overriden for custom configurations

    Default Implementation

    The default session will occur on the main queue and with its configuration. A default delegate is implemented.

    Declaration

    Swift

    var session: URLSession { get }
  • additionalHeaders Default implementation

    Defaults Headers, those headers can be provided when implementing the protocol. They will be added to each request

    Default Implementation

    Default headers should be left empty

    Declaration

    Swift

    var additionalHeaders: [String : String] { get }
  • additionalParams Default implementation

    Addional Parameters, those parameters can be provided when implementing the protocol. They will be added to each request

    Default Implementation

    Default params should be left empty

    Declaration

    Swift

    var additionalParams: [String : Any] { get }
  • validate(response:data:error:) Default implementation

    Validation function, it is called when a request is received. You can implement a custom validation if you want to define when we should consider the result a failure

    • paramater data: the data that was returned in the response’s body

    Default Implementation

    Default validation. Will return an Error if the HTTP status code is greater than 399 (400+: client errors, 500+: server errors)

    Declaration

    Swift

    func validate(response: HTTPURLResponse?, data: Data, error: Error?) -> MemoryChainError?

    Parameters

    response

    the HTTPURLResponse returned by NSURLSession
    error

    any error returned by NSURLSession

    Return Value

    any Error that you would like to pass to consider the result a failure

  • shouldContinue(with:) Default implementation

    A function called if an error is found before a request returns a failure response. It can be used for instance if the error is a HTTP 401 error and that you don’t want to proceed

    Default Implementation

    Default implementation of shouldContinue Will always return true. It is up to you whether or not you want to stop the request

    Declaration

    Swift

    func shouldContinue(with error: MemoryChainError) -> Bool

    Parameters

    error

    the error that the request currently holds

    Return Value

    a boolean that tells us whether or not we should send a result back

  • defaultEncoding(for:) Default implementation

    A function that allows you to define the default encoding for an HTTPMethod. By default this will return json for .post but you can implement that function to return .form instead

    Default Implementation

    Function that provides the default encoding for every HTTP method

    Declaration

    Swift

    func defaultEncoding(for method: HTTPMethod) -> ParameterEncoding

    Parameters

    method

    the HTTPMethod for which you need to define the encoding

    Return Value

    The proper encoding

    This is the method that needs to be called from a provider to send the request.

    Declaration

    Swift

    Parameters

    route

    the route object that defines the request

    Return Value
